Setup required

Finish your setup

Your app can’t start yet because it isn’t connected to Supabase. Supabase powers sign-in and the database, so it has to be wired up before any page can load. This takes a couple of minutes.

  1. Create a Supabase project Go to supabase.com/dashboard and create a new project (the free tier is fine). Note the database password you choose — you’ll need it below.
  2. Add the four Supabase variables to .env.local NEXT_PUBLIC_SUPABASE_URL NEXT_PUBLIC_SUPABASE_ANON_KEY SUPABASE_SERVICE_ROLE_KEY DATABASE_URL Find these in your Supabase dashboard. The project URL is on the project home header (or the top Connect button); the anon and service_role keys are under Settings → API Keys → “Legacy anon, service_role API keys”; and the DATABASE_URL is under Connect → Direct → Transaction pooler → URI (port 6543) — replace [YOUR-PASSWORD] with your DB password. Full paths are in the README Quickstart.
  3. Restart the dev server Stop and start it again so the new values load: 
    pnpm dev
Once Supabase is connected, this page disappears and the app loads normally. You can then sign up — the first account becomes the admin — and the in-app journey at /journey walks you through Stripe, Resend, branding, and deploy. Secrets stay in .env.local (gitignored) — never paste them into a web form.
